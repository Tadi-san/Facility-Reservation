# Eagle Platform Design

## Delivered Architecture
- Backend: Spring Boot 3 (Java 17), Spring Security, Spring Data JPA, PostgreSQL.
- Frontend: Angular application in `app/frontend` consuming Spring APIs.
- Docker verification path: `docker-compose.yml` runs PostgreSQL + backend + Angular frontend together.

## Core Security Model
- HTTP security is authenticated-by-default.
- JWT auth filter populates Spring Security context from `Authorization: Bearer`.
- Route-level RBAC:
  - `ADMIN`: user management, facilities, canary.
  - `OPS|ADMIN`: operations + finance management.
  - `AGENT|ADMIN`: front desk actions.
  - `TECH|ADMIN`: maintenance + assets.
- Object-level controls:
  - requesters can only operate on self-owned reservations unless `AGENT|ADMIN`.

## Required Controls
- Lockout policy: 5 failed attempts, 15-minute lock window (`User.lockUntil`).
- Password policy: 12+ chars with complexity checks.
- Immutable audit trail (`audit_logs`) with hash chaining and append-only writes.
- Sensitive field encryption at rest for `User.staffContactInfo` via AES-GCM JPA converter.
- Global exception handling with generic 500 responses and structured validation errors.

## Reliability and Operations
- Request rate limiting filter for API abuse reduction (60 requests/minute per authenticated user; IP fallback for unauthenticated calls).
- Durable async local notification queue (`notification_jobs`) with retries and circuit-open cooldown.
- Durable async search indexing queue using `notification_jobs` job type `REBUILD_SEARCH_INDEX`.
- Nightly encrypted backup snapshots (`backups/eagle-YYYY-MM-DD.enc`) generated by scheduler.
- Canary check endpoint (`GET /api/v1/health/canary`) for admin synthetic readiness checks.
- Static resource cache control headers configured on Spring static resources.
- Maintenance SLA automation: due time is computed as 4 business hours (UTC business window 09:00-17:00, weekdays), with scheduled overdue flagging.
- Critical asset inspection cadence: 30-day scheduler auto-creates inspection records for active critical assets.

## Feature Coverage
- Reservations: conflict detection, room reservability enforcement, check-in/checkout.
- Front desk: schedule board, handoffs, templates, async notification refresh.
- Maintenance: open/update/close ticket lifecycle with closure outcomes plus automated SLA due/overdue handling.
- Metrics: versioned templates persist weighted dimensions in structured JSON.
- Finance: offline orders with status lifecycle, idempotency keys, reconciliation with checksum duplicate detection, and reservation ownership enforcement for non-OPS/ADMIN users.
